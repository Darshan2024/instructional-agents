Instructional Goals Definition
==============================

1. Describe and compare various Software Development Life Cycle (SDLC) models, including Waterfall, Agile, and Spiral, and analyze their suitability for different project scenarios.  
2. Elicit, analyze, and document clear, complete software requirements using standard techniques and tools.  
3. Design software architectures employing modular design principles, and select appropriate design patterns to address specific project needs.  
4. Implement software using industry-relevant programming languages and tools, demonstrating proficiency in version control, IDEs, and debugging techniques.  
5. Apply testing methodologies, including unit, integration, and system testing, to verify software functionality and improve quality.  
6. Collaborate effectively in team-based software development projects, demonstrating communication, planning, and problem-solving skills.  
7. Identify ethical considerations and professional standards relevant to software development, and evaluate their implications in real-world scenarios.
