# 1. Course Description:
This course introduces fundamental concepts, methodologies, frameworks, and tools essential to software engineering. Students will explore SDLC models, requirements engineering, system and software design principles, development practices, testing methodologies, team collaboration, and professional ethics, preparing them for real-world software development scenarios. Emphasis is placed on industry relevance, practical skills, ethical considerations, and professional communication.

# 2. Course Learning Objectives:
Upon successful completion, students will be able to:
- Describe and compare SDLC models (Waterfall, Agile, Spiral) and evaluate their suitability for different project contexts.
- Elicit, analyze, and document software requirements using industry-standard techniques.
- Model requirements with UML and other tools to support system design.
- Design modular, scalable software architectures employing common design patterns.
- Implement software solutions using relevant programming languages and tools; demonstrate effective version control and debugging skills.
- Apply testing methodologies (unit, integration, system) and utilize testing tools to ensure software quality.
- Collaborate effectively in team-based development environments, demonstrating communication, planning, and project management skills.
- Evaluate ethical issues, professional standards, and societal impacts related to software engineering practice.
- Demonstrate professional communication and documentation skills suitable for industry settings.

# 3. Weekly Schedule & Topics

| Week | Topics | Learning Objectives | Required Readings | Assessments |
|-------|---------------------------------------------------------|---------------------------------------------------------|-------------------------------------------------------------------|------------------------------|
| 1 | **Introduction & Course Orientation** | Understand course structure, resources, tools; assess prior knowledge and expectations | Course syllabus; introductory materials | Participation; baseline assessment quiz |
| 2 | **Software Engineering & SDLC Overview** | Describe SDLC models (Waterfall, Agile, Spiral); compare their applicability | Pressman’s Software Engineering, Ch. 1 & 2 | Quiz on SDLC concepts |
| 3 | **Requirements Engineering** | Elicit, analyze, document requirements professionally; apply standard techniques | Wiegers’ Software Requirements, Ch. 1-3 | Requirements documentation assignment |
| 4 | **Requirements Modeling & Visual Tools** | Use UML and modeling tools to represent requirements; develop models | UML Distilled; Sommerville, Ch. 5 | Modeling exercise with UML diagrams |
| 5 | **Software Architecture & Design Principles** | Design modular, scalable architectures; employ design principles | Gamma et al., Design Patterns (selected chapters) | Architecture design draft presentation |
| 6 | **Design Patterns in Practice** | Select appropriate design patterns for project needs; evaluate pattern impacts | Refactoring Guru website; case studies | Case study analysis report |
| 7 | **Development Environments & Version Control** | Use IDEs (Eclipse, Visual Studio); implement version control with Git | Git official documentation; IDE tutorials | Mini-project on version control setup |
| 8 | **Implementation Techniques & Debugging** | Develop code with best practices; utilize debugging tools effectively | Language-specific tutorials (Java/Python) | Coding assignment demonstrating debugging skills |
| 9 | **Testing Methodologies: Unit, Integration, System** | Plan and execute tests using frameworks (JUnit, pytest); write effective test cases | Pressman, Ch. 6 | Test cases documentation and execution report |
| 10 | **Automation & Test Management Tools** | Use Jenkins, Selenium for automation; manage test execution | Introductory tutorials; online resources | Demo of test automation implementation |
| 11 | **Team Collaboration & Agile Practices** | Collaborate via Jira, Trello; understand Scrum and Kanban workflows | Agile Manifesto; Scrum Guide | Group project planning session and backlog management |
| 12 | **Project Management & Effective Communication** | Document progress; communicate clearly; manage project scope and risks | "Effective Project Management" guide | Peer review of project documentation and progress report |
| 13 | **Professional Ethics & Standards** | Analyze ethical dilemmas; apply ACM/IEEE Code of Ethics | ACM/IEEE Code of Ethics; relevant case studies | Ethical scenario analysis and reflection |
| 14 | **Course Project Presentations & Wrap-up** | Present final projects; synthesize learning; receive peer and instructor feedback | All course materials | Final project presentation; peer and instructor evaluations |

# 4. Assessment & Grading Policy

|------------------------|--------------|------------------|------------------------|
