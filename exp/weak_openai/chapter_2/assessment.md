  B) It speeds up development without planning.

**Correct Answer:** C
**Explanation:** Following a structured SDLC helps manage project complexity, facilitates quality control, and reduces the likelihood of errors.

## Section 3: Waterfall Model

### Learning Objectives
- Describe the phases of the Waterfall model.
- Discuss the advantages and limitations of the Waterfall approach.
- Identify appropriate use cases for the Waterfall model.

### Assessment Questions

**Question 1:** What is a primary advantage of the Waterfall model?

  A) Flexibility to changes late in development
  B) Clear, structured phases that are easy to manage
  C) Iterative feedback and revisions
  D) Emphasizes stakeholder involvement throughout

**Correct Answer:** B
**Explanation:** The Waterfall model's structured, sequential phases make project management straightforward, though it lacks flexibility.

**Question 2:** Which phase in the Waterfall model involves designing the system architecture based on requirements?

  A) Implementation
  B) Requirement Analysis
  C) System Design
  D) Deployment

**Correct Answer:** C
**Explanation:** System Design involves creating architecture and specifications based on the gathered requirements.

**Question 3:** Which of the following is a common limitation of the Waterfall model?

  A) It allows excessive flexibility that can delay projects
  B) Testing occurs early, often before implementation
  C) It is inflexible and difficult to accommodate changes after initial phases
  D) It neglects documentation and planning steps

**Correct Answer:** C
**Explanation:** A key limitation is its inflexibility; once a phase is completed, revisiting it is difficult, making accommodating change challenging.

**Question 4:** In which scenarios is the Waterfall model most appropriately used?

  A) Projects with rapidly changing requirements
  B) Large-scale social media app development
  C) Embedded system development with stable requirements
  D) Iterative game design

**Correct Answer:** C
**Explanation:** The Waterfall model is suitable for projects with well-understood, stable requirements like embedded systems.

## Section 5: Spiral Model

### Learning Objectives
- Explain the core concept of the Spiral model.
- Identify how risk management influences its iterative process.

### Assessment Questions

**Question 1:** What is the main focus of the Spiral model?

  A) Rapid delivery of a fully functional product
  B) Strict sequential phases without iteration
  C) Managing risk through iterative refinement
  D) Eliminating documentation

**Correct Answer:** C
**Explanation:** The Spiral model emphasizes risk management by iterative risk analysis and refinement during development.

**Question 2:** Which phase is NOT explicitly part of each spiral iteration?

  A) Planning
  B) Risk Analysis
  C) Coding and Implementation
  D) Evaluation

**Correct Answer:** C
**Explanation:** While coding can occur during engineering, the explicit phases of each spiral are Planning, Risk Analysis, Engineering (which includes development), and Evaluation. Coding is part of engineering but not labeled as a separate phase in the description.

**Question 3:** In what types of projects is the Spiral model most appropriate?

  A) Small, well-defined projects with low risk
  B) Projects with high complexity or high risk
  C) Projects following strict linear development
  D) Projects with very short timelines

**Correct Answer:** B
**Explanation:** The Spiral model is designed to handle complex, high-risk projects where risk management and iterative refinement are critical.

### Activities
- Draw a diagram of the Spiral model illustrating the phases: Planning, Risk Analysis, Engineering, and Evaluation. Indicate how each loop expands or refines the project, emphasizing risk assessment integration.

### Discussion Questions
- How does prioritizing risk management in the Spiral model compare to traditional linear models in high-risk projects?
- Can the Spiral model be adapted for smaller projects? Why or why not?

**Question 1:** Which of the following best describes the primary difference between Waterfall and Agile SDLC models?

  A) Waterfall is iterative, while Agile is sequential.
  B) Waterfall emphasizes flexibility, whereas Agile emphasizes predictability.
  C) Waterfall follows a linear, sequential process, while Agile promotes iterative development and frequent feedback.
  D) Waterfall requires active stakeholder involvement at every stage, unlike Agile.

**Correct Answer:** C
**Explanation:** Waterfall is a linear, sequential model where each phase completes before the next begins, whereas Agile is an iterative model emphasizing cycles of development with ongoing stakeholder feedback.

**Question 2:** Which factor is most likely to favor the use of an agile SDLC model over a waterfall model?

  A) Strict regulatory requirements with well-defined processes
  B) A project with rapidly changing features and evolving requirements
  C) Very large, fixed scope projects with fixed delivery dates
  D) Projects where detailed documentation is the top priority

**Correct Answer:** B
**Explanation:** Agile is well-suited for projects with changing requirements and the need for flexibility, enabling rapid adaptations and continuous feedback.

**Question 3:** In choosing an SDLC model, which of the following is NOT a common consideration?

  A) Project complexity and size
  B) Team's preferred programming language
  C) Requirement stability and clarity
  D) Stakeholder engagement level

**Correct Answer:** B
**Explanation:** While programming language is important for development, it is NOT a primary factor in selecting an SDLC model; considerations focus more on project scope, requirements, and stakeholder involvement.

**Question 4:** Why is it beneficial to ask questions about SDLC choices during project planning?

  A) To satisfy management documentation requirements
  B) To clarify understanding, mitigate risks, and make informed decisions about process selection
  C) To delay project starting times

**Correct Answer:** B
**Explanation:** Asking questions helps clarify requirements, understand trade-offs, and ensure the chosen SDLC aligns with project goals, reducing risks and increasing success chances.
