Final Project Assessment Design
===============================

# Final Project Plan: Industry-Driven Software System Development

## 1. Project Description
This capstone project immerses students in authentic software engineering practices by developing a stakeholder-informed software solution addressing a realistic scenario—such as a requirements management portal or project management tool. Students will undergo the entire Software Development Life Cycle (SDLC), from requirements gathering and system design to implementation, testing, and ethical evaluation. The project emphasizes applied skills, professional communication, collaboration, and ethical reasoning, reflecting industry standards and fostering higher-order thinking.

## 2. Objectives
- Demonstrate comprehensive understanding of SDLC models and methodologies.
- Effectively gather, analyze, and model requirements using UML and other tools.
- Develop a modular, well-architected software system adhering to design best practices.
- Implement and rigorously test a functional software product.
- Document all phases with clarity, professionalism, and ethical consideration.
- Collaborate efficiently within teams, providing and receiving constructive feedback.
- Reflect critically on technical decisions, challenges, and ethical responsibilities.

## 3. Timeline & Milestones
| Milestone | Description | Due Date | Format & Submission Platform | Purpose/Assessment Focus |
|--------------|--------------------------|--------------|-------------------------------------|------------------------------|
| 1. Project Proposal & Scope | Define problem, stakeholders, objectives, initial ideas | Week 2 | PDF via Canvas | Clarity, understanding, feasibility |
| 2. Requirements & Modeling Report | Gather, analyze, document requirements + UML diagrams | Week 4 | PDF + UML files uploaded to Canvas folder | Completeness, modeling skills |
| 3. Architecture & Design Draft | System architecture, component diagrams, design patterns | Week 6 | PDF + UML files | Design reasoning, architectural soundness |
| 4. Implementation Progress & Challenges | Summarize development, obstacles, insights | Week 8 | PDF report | Reflection, problem-solving |
| 5. Testing & Quality Assurance Report | Testing plans, results, bug logs | Week 10 | PDF + link to code repository | Testing rigor, quality focus |
| 6. Final Deliverable & Reflection | Complete system, documentation, team reflection | Week 12 | Zip of repo + PDFs + optional video | Functionality, professionalism, ethical reflection |

## 4. Final Deliverables
- **Working Software:** Fully implemented and tested system, with setup instructions.
- **Requirements & Design Artifacts:** Requirements specification, UML diagrams, architecture diagrams.
- **Testing Documentation:** Test cases, coverage reports, bug logs.
- **Reflection Paper:** Personal or team reflection discussing processes, challenges, ethical considerations.
- **Peer Feedback:** Optional structured peer review forms assessing collaboration.
- **Presentation (Optional):** Short recorded or live presentation summarizing the project.

**Supported formats:** 
- Code repositories (GitHub/GitLab link)
- Documentation (.pdf, .md)
- Diagrams (.png, .uml)
- Videos (.mp4) or slides (.pdf)

## 5. Team & Individual Components
- **Teams:** Collaborative development, communication, and project management.
- **Optional Individual Work:** Reflection papers, ethical analyses, peer reviews.

## 6. Assessment & Rubrics
| Criteria | Weight | Description |
|--------------|--------------|---------------------------|
| Technical completeness & correctness | 35% | Functionality, adherence to requirements |
| Design quality & architecture | 15% | Architectural decisions, design patterns |
| Testing coverage & quality | 15% | Test cases, bug fixing, robustness |
| Documentation clarity & professionalism | 10% | Clarity, organization, professionalism |
| Reflection depth & ethical reasoning | 10% | Thoughtfulness, ethical considerations |
| Team collaboration & peer feedback | 10% | Contribution, communication, feedback quality |

Rubrics are explicitly aligned with course objectives, emphasizing critical thinking, ethical reasoning, technical skills, and communication.

## 7. Feedback & Checkpoints
- Regular instructor and peer feedback after each milestone via Canvas.
- Mid-project peer review to promote self and peer assessment.
- Scheduled check-ins and optional workshops at each phase for support.
- Opportunities for extensions or flexible deadlines if needed.

## 8. Academic Integrity & Ethical Guidelines
- Individual accountability within team projects.
- Original work, with originality verification via version control and plagiarism checks.
- Using authentic, publicly available datasets when applicable.
- Reflection papers include discussions on professional standards, societal impacts, and ethical dilemmas.
- Proper citation and acknowledgment of sources.
- Maintain confidentiality and respect privacy/data protection standards.

## 9. Rationale & Educational Value
This structured, iterative project fosters authentic learning, simulating real-world software development. It encourages higher-order reasoning, reflection, and ethical practice. The milestones provide continuous formative assessment and feedback, supporting skill development and ensuring equitable participation. Incorporating real-world scenarios promotes relevance and engagement across diverse learner backgrounds.
