Assessment & Evaluation Planning
================================

Assessment and Evaluation Plan for Software Engineering 1

I. Core Components of the Assessment Strategy
1. Milestone-Based Deliverables
2. Project-Based Final Deliverable
3. Continuous & Formative Assessments
4. Reflection & Ethical Analysis
5. Peer & Instructor Feedback
6. Participation & Engagement

II. Detailed Breakdown

A. Milestones & Interim Deliverables

| Milestone | Description | Format | Submission Platform & Format | Grading Criteria |
|-------------|-------------|--------|------------------------------|------------------|
| 1. Project Proposal | Define scope, objectives, initial requirements | Open-ended document, diagrams | PDF upload via Canvas | Clarity, feasibility, understanding of project context |
| 2. Requirements & Modeling Report | Elicit, analyze, model requirements using UML | PDF report + UML diagram files (.uml, .png) | PDF + UML files in Canvas | Completeness, accuracy, standards compliance, clarity of models |
| 3. Architecture & Design Draft | Present system architecture, design patterns | PDF + UML diagrams (class/sequence) | PDF + UML files | Appropriateness of design, technical soundness |
| 4. Implementation Progress Report | Summarize development status, challenges | PDF | PDF upload | Reflection depth, problem-solving insight |
| 5. Testing & Quality Assurance Report | Document testing strategies, results | PDF + code snippets or links | PDF + code ZIP or repository link | Test coverage, thoroughness, correctness |
| 6. Final Project Submission | Complete working system + documentation | ZIP archive, Git repo link, PDFs, presentation media | Canvas folder upload | Functionality, code quality, documentation clarity, adherence to standards |

B. Capstone Project (Due at end of term)
- Students form teams to develop a real-world-inspired software solution (e.g., project management tool, requirements portal, simple e-commerce system).
- Components:
  - Requirements Specification (documented stakeholder needs)
  - System Design & Architecture (UML diagrams, pattern choices)
  - Implementation (functional software)
  - Testing (unit, integration, system testing)
  - Documentation (user manual, developer notes)
  - Reflection & Lessons Learned
  - Team Collaboration & Presentation
- **Submission Formats:**
  - Code: .zip archive or Git repository link
  - Documentation: PDF (preferred) or Markdown (.md)
  - Presentation/Video: .mp4 or .pdf
- **Rubric (100 points total):**
  - Requirements clarity and completeness (20)
  - Architectural soundness (15)
  - Code quality and functionality (25)
  - Testing thoroughness (15)
  - Documentation and professionalism (15)
  - Oral presentation and reflection (10)

C. Continuous & Formative Assessments
- **Open-Ended Questions (Weekly/Bi-weekly):**  
  Scenario analyses, design choices, ethical dilemmas.  
  *Format:* Canvas quizzes or assignment uploads (.pdf/.docx).  
  *Rubric:* Critical thinking, relevance, evidence-based reasoning.
- **Multiple-Choice Questions (MCQs):**  
  Focused on SDLC models, testing methodologies, foundational principles.  
  *Format:* Canvas quizzes.  
  *Rubric:* Accuracy, conceptual understanding.
- **Participation & Engagement:**  
  Active involvement in discussions, peer reviews, workshops.  
  *Assessment:* Participation logs, peer feedback forms.

D. Reflection & Ethical Considerations
- **Individual Reflection Papers:**  
  Post-milestone or project completion reflections on challenges, decisions, professional ethics.  
  *Format:* PDF or Word document.  
  *Purpose:* Foster critical thinking about professional standards and societal impacts.

III. Grading Rubrics (Sample)
- Clarity & Completeness (20 points): Clear articulation, thorough coverage.
- Technical Accuracy (20 points): Correct use of design principles, coding standards.
- Professional Presentation (15 points): Organized, error-free documents, effective visuals.
- Creativity & Problem-Solving (15 points): Innovative solutions, thoughtful design.
- Team Collaboration & Communication (15 points): Effective teamwork, conflict resolution evidence.
- Ethical Consideration (10 points): Depth of analysis, professional rigor.

IV. Submission & Logistics
- All documents and code to be uploaded via Canvas in specified formats.
- Milestones facilitate iterative feedback to encourage continuous improvement.
- Final presentations may be recorded and shared for peer review.

V. Rationale & Emphasis
- Emulates industry software development lifecycles via projects.
- Milestones support progress tracking and formative feedback.
- Diverse assessment types foster deep understanding beyond rote memorization.
- Capstone integrates skills, emphasizing practical application, teamwork, and ethics.
- Avoids heavy reliance on traditional exams, promoting engagement, reflection, and readiness for the software industry.
