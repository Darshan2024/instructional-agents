Final Project Assessment Design
===============================

# Final Project Plan: Software Development Lifecycle Simulation Project

## Project Overview
The capstone project will simulate a real-world software development scenario, allowing students to collaboratively define, design, develop, and deliver a software product, demonstrating their understanding of software engineering principles, agile methodologies, programming skills, testing practices, ethical considerations, and technical communication.

---

## Objectives
By completing this project, students will:
1. Apply software development lifecycle methodologies (Agile, Waterfall) to a real-world project.
2. Develop a fully functional software application using object-oriented programming principles.
3. Utilize algorithms and data structures to solve specific problems within their project.
4. Collaborate effectively using version control (Git) and project management tools.
5. Conduct unit and integration tests for quality assurance on their software.
6. Analyze ethical considerations in software development and document implications.
7. Produce comprehensive technical documentation suitable for both technical and non-technical stakeholders.

---

## Timeline with Milestones

| Milestone                                | Due Date | Format                                                    | Content                                                                          | Weight |
|------------------------------------------|----------|----------------------------------------------------------|----------------------------------------------------------------------------------|-------|
| Milestone 1: Project Proposal            | Week 5   | Written proposal (.pdf)                                  | A detailed description including proposed software features, methodology, roles, and timeline. | 10%   |
| Milestone 2: Progress Update             | Week 9   | Progress report including Git commit history (.pdf)     | Document work completed, challenges encountered, and future pivot strategies.    | 15%   |
| Milestone 3: Draft Technical Documentation| Week 10  | Draft documentation (.pdf)                              | Draft covering installation instructions, API usage, and user manuals.         | 10%   |
| Milestone 4: Final Software Product      | Week 12  | Fully functional software product (.ipynb) + presentation (.pdf) | An application meeting proposal specifications and a final presentation.       | 35%   |
| Milestone 5: Ethical Reflection Paper    | Week 9   | Written reflection (.pdf)                                | Critical analysis of ethical dilemmas faced using course materials.              | 10%   |

---

## Deliverables
- GitHub repository link containing project code.
- PDF documents of project proposal, progress update, draft documentation, ethical reflection, and final presentation.
- Fully functional software product in Jupyter Notebook format.

---

## Grading Rubric
Each milestone will be evaluated based on:
1. Content clarity
2. Technical accuracy
3. Collaboration evidence
4. Completion of tasks
5. Student engagement with software engineering principles

---

## Submission Formats
- All submissions must be in .pdf and .ipynb formats, with the exception of the GitHub link.
- Late submissions incur a penalty of 10% of the grade per day without prior arrangements.

---

## Academic Integrity Guidelines
- All projects must be original work from the submitting team.
- Plagiarism checks will be conducted using approved software.
- Collaboration is encouraged; however, each student must contribute fairly to their team.
- Creation of a group contract outlining responsibilities is encouraged to ensure fair collaboration.

--- 

## Conclusion
This capstone project aims to provide students with a practical application of their knowledge gained throughout the course, culminating in a simulation of real-world software engineering environments while fostering critical thinking and teamwork skills.
