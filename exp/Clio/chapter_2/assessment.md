  A) To learn programming languages

  A) Waterfall

  C) DevOps
  D) Spiral

  B) By conducting testing at the end of the cycle
  C) By ensuring thorough documentation before progressing
  D) By incorporating feedback after each iteration

**Correct Answer:** C
**Explanation:** Waterfall methodology uses a sequential approach to ensure thorough documentation and design before coding to reduce risk.

  A) It allows for personal preferences in coding styles
  B) It fosters better communication and understanding

**Explanation:** A common methodology fosters better communication as all team members are aligned on processes and expectations.

  D) Completed software code

  A) Flexible to changes during development
  B) Well-defined stages with specific deliverables
  C) No need for documentation
  D) Can handle vague requirements

**Correct Answer:** B
**Explanation:** The Waterfall methodology is well-structured, with clear phases that have specific deliverables and a review process.

## Section 5: Scrum Framework

### Learning Objectives
- Identify the key roles in the Scrum framework.
- Explain the events and artifacts associated with Scrum.
- Describe the purpose of each Scrum event and the responsibilities of each role.

### Assessment Questions

**Question 1:** What is the primary responsibility of the Product Owner?

  A) To oversee the daily Scrum meetings
  B) To maximize the value of the product
  C) To manage the development team directly
  D) To ensure all tasks are completed on time

**Correct Answer:** B
**Explanation:** The Product Owner is responsible for maximizing the value of the product and managing the product backlog.

**Question 2:** Which of the following is NOT an event in the Scrum framework?

  A) Sprint Review
  B) Daily Scrum
  C) Sprint Planning
  D) Product Backlog Grooming

**Correct Answer:** D
**Explanation:** Product Backlog Grooming is not formally recognized as an event in Scrum, although it may occur as part of backlog refinement.

**Question 3:** What is the purpose of the Sprint Retrospective?

  A) To plan the next Sprint's tasks
  B) To review the increment created during the Sprint
  C) To identify opportunities for process improvement
  D) To showcase the product to stakeholders

**Correct Answer:** C
**Explanation:** The Sprint Retrospective is an opportunity for the team to reflect on the past Sprint and identify areas for improvement.

**Question 4:** Which artifact provides an ordered list of everything needed in the product?

  A) Sprint Backlog
  B) Increment
  C) Product Backlog
  D) Definition of Done

**Correct Answer:** C
**Explanation:** The Product Backlog is an ordered list of everything needed in the product, and it evolves as the project progresses.

**Question 1:** When is it most appropriate to use the Waterfall methodology?

  A) When requirements are unclear

  C) In dynamic environments
  D) For startups

  B) An approach that encourages rapid delivery and flexibility
  C) A method that prioritizes extensive documentation
  D) A process that avoids stakeholder feedback

**Explanation:** Agile methodologies focus on iterative development, rapid delivery, and adapting to change.

**Question 3:** What is a key characteristic of Waterfall methodology?

  A) Continuous customer collaboration
  B) Phased project stages without overlap
  C) Minimal documentation
  D) Flexibility to accommodate changes

  A) Strict adherence to initial project plans
