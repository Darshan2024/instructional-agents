Assessment & Evaluation Planning
================================

# Assessment and Evaluation Plan for Software Engineering 1

## Overview
This assessment plan employs project-based evaluations and milestone-driven assessments to enhance real-world application and analytical thinking among students. The structure centers around a final capstone project, accompanied by smaller projects, proposals, and peer-reviewed presentations.

---

## Assessments Breakdown

### 1. Milestone Assessments
#### Milestone 1: Group Project Proposal (10%)
- **Format:** Written proposal (.pdf)
- **Content:** Description of proposed project, software engineering methodologies, team roles, and timeline.
- **Due Date:** Week 5
- **Grading Rubric:**
  - Clarity and conciseness of proposal (20%)
  - Depth of research & understanding of methodologies (30%)
  - Feasibility and planning (30%)
  - Team roles and responsibilities (20%)

#### Milestone 2: Progress Report (15%)
- **Format:** Progress report (.pdf)
- **Content:** Document progress on the project, including completed tasks, challenges faced, and pivots made; Git commit history snapshot (.ipynb or .pdf).
- **Due Date:** Week 9
- **Grading Rubric:**
  - Completeness of reported progress and challenges faced (40%)
  - Evidence of collaboration using version control (30%)
  - Quality of documentation (30%)

#### Milestone 3: Draft Technical Documentation (10%)
- **Format:** Technical document (.pdf)
- **Content:** Draft of the technical documentation, including API use, installation process, and user guides.
- **Due Date:** Week 10
- **Grading Rubric:**
  - Clarity and organization (40%)
  - Completeness of documentation (30%)
  - Ability to convey technical concepts to non-technical users (30%)

### 2. Practical Programming Projects (30%)
#### Project 1: Object-Oriented Programming Assignment (15%)
- **Format:** Code repository submission on GitHub (.ipynb)
- **Due Date:** Week 3
- **Grading Rubric:**
  - Code efficiency and performance (40%)
  - Adherence to object-oriented principles (40%)
  - Proper use of version control (20%)

#### Project 2: Testing and Quality Assurance Assignment (15%)
- **Format:** Code repository submission on GitHub (.ipynb)
- **Due Date:** Week 8
- **Grading Rubric:**
  - Coverage and completeness of tests (50%)
  - Ability to identify and fix bugs (30%)
  - Documentation of testing process (20%)

### 3. Reflection Paper on Ethical Dilemma (10%)
- **Format:** Written reflection (.pdf)
- **Content:** Analyze a real-world ethical dilemma faced in software engineering using class materials and discussions.
- **Due Date:** Week 9
- **Grading Rubric:**
  - Insightfulness and critical thinking (40%)
  - Connection to ethical frameworks discussed in class (30%)
  - Clarity of writing and argument structure (30%)

### 4. Final Project (35%)
- **Format:** Group project culminating in presentation and final software product (.ipynb, .pdf)
- **Content:** Deliver a fully functional software product based on the proposed project idea, including documentation and a presentation.
- **Due Date:** Week 12
- **Grading Rubric:**
  - Functionality and performance of the software (40%)
  - Quality of presentation (30%)
  - Collaboration and teamwork dynamics (20%)
  - Documentation quality (10%)

---

## Submission Logistics
- **Final Submission Platform:** Canvas LMS
- **File Formats Required:**
  - Written documents: .pdf
  - Code submissions: .ipynb via GitHub (link shared in Canvas)
- **Late Submission Policy:** 10% penalty per day unless prior arrangements are made with the instructor.

---

## Scheduling and Communication
- **Regular Feedback Sessions:** Continuous feedback on proposals and milestones throughout the course to foster iterative learning.
- **Office Hours:** Weekly to address questions related to assignments, project progress, and collaboration challenges.
- **Collaboration Tools:** Slack or Microsoft Teams for team communication.

---

## Conclusion
This multifaceted assessment strategy in the Software Engineering 1 course enhances student learning through real-world engagement and collaborative projects, focusing on practical applications and interpersonal skills necessary for future careers.
