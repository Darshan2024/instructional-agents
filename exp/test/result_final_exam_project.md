Final Project Assessment Design
===============================

# Final Project Plan for Python Fundamentals Capstone Project

## Project Description
The Capstone Project for the Python Fundamentals course involves the development of a Python application that addresses a real-world problem. Students will work in teams or individually to apply their knowledge of Python syntax, data structures, external libraries, debugging, and testing in a collaborative environment.

## Project Objectives
1. Identify and solve a real-world problem using Python.
2. Integrate the use of external libraries for data manipulation and visualization.
3. Enhance collaboration and communication skills.
4. Foster critical thinking and analytical problem-solving abilities.

## Timeline with Milestones

| Milestone                  | Description                                                 | Due Date   | Assessment Weight |
|---------------------------|-------------------------------------------------------------|------------|-------------------|
| **Project Proposal**      | Written proposal (2-3 pages) laying out project scope and objectives. | Week 6    | 5%                |
| **Progress Report**       | Update (1-2 pages) on project status, methodologies, and challenges encountered. | Week 10   | 5%                |
| **Development Phase**     | Working prototype of the application presented in a coding review. | Weeks 11-12 | 10%               |
| **Final Presentation**     | Comprehensive report (5-10 pages) and 15-minute oral presentation, including final application code. | Weeks 14-15 | 15%               |

## Deliverables
1. **Project Proposal**: PDF submitted via Canvas.
2. **Progress Report**: PDF submitted via Canvas.
3. **Development Phase**: Working code repository on GitHub with README.
4. **Final Report and Presentation**: PDF report and presentation slides submitted via Canvas; code repository remains open.

## Grading Rubric
- **Project Proposal**: 
  - Clarity (1 point)
  - Feasibility (1 point)
  - Depth of analysis (1 point)
  - Engagement with feedback (2 points)

- **Progress Report**:
  - Clarity (2 points)
  - Analysis (2 points)
  - Incorporation of feedback (1 point)

- **Development Phase**:
  - Functionality (5 points)
  - Complexity (3 points)
  - Documentation (2 points)

- **Final Presentation**:
  - Functionality (30 points)
  - Code Complexity (20 points)
  - Collaborative Efforts (20 points)
  - Presentation Quality (30 points)

## Submission Formats
- **Project Proposal**: PDF via Canvas
- **Progress Report**: PDF via Canvas
- **Development Phase Submission**: GitHub repository
- **Final Report and Presentation**: PDF via Canvas and GitHub repository

## Academic Integrity Guidelines
- Emphasize original work while allowing collaboration.
- All students must sign an academic integrity statement with the final submission.
- Implement peer evaluations to ensure accountability in group contributions.

## Additional Notes
- Weekly check-ins are encouraged to facilitate progress and provide feedback.
- Structured peer reviews will guide feedback quality and support learning.
- Additional resources, including workshops and literature reviews, will be provided to support student understanding and integrity in coding practices.
