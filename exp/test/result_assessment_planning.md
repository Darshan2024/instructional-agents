Assessment & Evaluation Planning
================================

# Assessment and Evaluation Plan for Python Fundamentals Course

## Overview
This plan outlines the assessment strategy designed to promote project-based learning, enhancing real-world application and analytical thinking in Python programming.

---

## Assessment Components

### 1. Weekly Coding Assignments (30%)
- **Format**: Hands-on coding exercises relevant to weekly topics.
- **Timing**: Submitted at the end of each week (Fridays).
- **Submission Format**: .py files or .ipynb via Canvas LMS.
- **Grading Rubric**:
  - **Correctness (50 points)**: Code functions as intended.
  - **Efficiency (20 points)**: Use of optimal algorithms and data structures.
  - **Code Style (20 points)**: Clarity, organization, adherence to Python PEP 8 style guide.
  - **Documentation (10 points)**: Use of comments and docstrings.

### 2. Milestone Progress Reports (20%)
- **Format**: Written reports on the Capstone Project, including problem statements, methodologies, and preliminary results.
- **Timing**:
  - **Proposal (5%)**: Week 6.
  - **Progress Report (5%)**: Week 10.
  - **Final Reflection (10%)**: Week 14.
- **Submission Format**: .pdf via Canvas LMS.
- **Grading Rubric**:
  - **Clarity of Proposal/Report (25 points)**: Logical coherence, organization.
  - **Depth of Analysis (25 points)**: Understanding of concepts applied.
  - **Professional Presentation (25 points)**: Formatting and overall aesthetics.
  - **Engagement with Feedback (25 points)**: Incorporation of peer and instructor feedback.

### 3. Capstone Project (30%)
- **Format**: Group project applying course concepts to solve a real-world problem using Python.
- **Timing**:
  - **Proposal (5%)**: Week 6.
  - **Development Phase (10%)**: Week 12.
  - **Final Presentation (15%)**: Week 14-15.
- **Submission Format**: Project code repository (e.g., GitHub) and presentation slides (.pdf).
- **Grading Rubric**: 
  - **Functionality (30 points)**: Code operates as described in project scope.
  - **Complexity (20 points)**: Use of diverse data structures and algorithms.
  - **Collaboration (20 points)**: Contributions from all group members; peer evaluations included.
  - **Presentation Quality (30 points)**: Clarity of delivery, effective visuals, engagement with audience.

### 4. Participation and Engagement (10%)
- **Format**: Active participation in class discussions, peer reviews, and collaborative exercises.
- **Timing**: Assessed weekly.
- **Grading Method**: Self-evaluations and peer evaluations.
- **Grading Rubric**:
  - **Contribution to Discussions (50 points)**: Active voice in class; offers constructive feedback.
  - **Collaboration (50 points)**: Willingness to help peers; contributions to group work.

### 5. Self-Reflection Journals (10%)
- **Format**: Regular written entries reflecting on learning experiences and growth.
- **Timing**: Bi-weekly submissions (Weeks 2-14).
- **Submission Format**: .pdf or .docx via Canvas LMS.
- **Grading Rubric**:
  - **Insights Gained (35 points)**: Depth of reflection on personal learning.
  - **Problematic Areas (35 points)**: Articulated challenges faced.
  - **Forward Planning (30 points)**: Plans for continued learning.

---

## Question Types
- **Open-Ended Questions**: Used in weekly assignments and self-reflection journals.
- **Peer Review Questions**: Guided questions for peer assessments in Capstone Projects.

---

## Submission Logistics
- All submissions via Canvas LMS with clear deadlines.
- A late submission policy detailing penalties will be employed.
- Technical requirement guidance will be offered in an initial workshop.

---

## Evaluation Schedule
- Assignments graded within one week with detailed feedback.
- Participation feedback bi-weekly.

---

## Cumulative or Staged Final Project
- The Capstone Project replaces the traditional final exam, integrating all course content.
- Final presentations assessed in the last weeks of the course.

---

## Real-World Applications
- Assessments will connect classroom learning with practical scenarios through actionable projects and collaborative work, enhancing student engagement and applicability of skills in the industry. 

--- 

This assessment and evaluation strategy aims to empower students with the skills necessary to thrive in a rapidly evolving tech environment, preparing them for both immediate challenges and long-term career success.
