## Section 1: Introduction to Capstone Project Development

### Learning Objectives
- Understand the significance of the capstone project in relation to course objectives.
- Identify key areas of focus during project development and execution.

### Assessment Questions

**Question 1:** What is the purpose of the capstone project in the Python Fundamentals course?

  A) To learn new programming languages
  B) To develop a portfolio piece
  C) To enhance team skills
  D) To pass the final exam

**Correct Answer:** B
**Explanation:** The capstone project serves as a practical application of skills learned throughout the course, culminating in a portfolio piece.

**Question 2:** Which skill is primarily demonstrated in the capstone project?

  A) Understanding theoretical concepts
  B) Competence in a single topic
  C) Project management and teamwork
  D) Language syntax knowledge

**Correct Answer:** C
**Explanation:** The capstone project emphasizes project management and teamwork, allowing students to apply their skills in a collaborative environment.

**Question 3:** What is an example of a problem a student might address in their capstone project?

  A) Creating static websites
  B) Sorting large datasets
  C) Writing a research paper
  D) Memorizing programming syntax

**Correct Answer:** B
**Explanation:** Students may choose to create programs that automate processes like sorting datasets, which allows them to address real-world problems.

**Question 4:** What is a benefit of collaboration for capstone projects?

  A) Reducing the workload for each member
  B) Learning to divide tasks based on strengths
  C) Avoiding mistakes in coding
  D) Holding individual accountability

**Correct Answer:** B
**Explanation:** Collaboration allows students to learn effective communication and how to divide tasks based on the strengths of team members.

### Activities

### Learning Objectives
- Describe the expectations for the development phase.
- Engage in coding reviews to enhance project quality.

  A) Submit a final document without feedback
  B) Ignore coding reviews
  C) Create a final prototype and participate in reviews
  D) Work individually without collaboration

**Explanation:** Students are expected to actively participate in the development phase by creating prototypes and engaging in coding reviews.

  B) Leave code comments for clarity
  C) Avoid using version control
  D) Write code only for yourself

  C) Only integration tests
  D) Both unit and integration tests
