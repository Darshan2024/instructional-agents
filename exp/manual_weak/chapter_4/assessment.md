## Section 5: Algorithm Design Techniques

### Learning Objectives
- Explain common algorithm design techniques.
- Illustrate how different techniques differ from each other.
- Apply algorithm design techniques to solve computational problems.

### Assessment Questions

**Question 1:** Which of the following is a design technique that builds solutions incrementally?

  A) Greedy algorithms
  B) Brute-force
  C) Heuristics
  D) None of the above

**Correct Answer:** A
**Explanation:** Greedy algorithms build solutions incrementally by making the best choice at each step.

**Question 2:** What is the main characteristic of Dynamic Programming?

  A) It tries every possible combination without regard for efficiency.
  B) It solves each subproblem only once and uses stored results.
  C) It selects the first available option that seems best.
  D) It never uses recursion.

**Correct Answer:** B
**Explanation:** Dynamic Programming efficiently solves problems by breaking them down into smaller overlapping subproblems and storing their solutions.

**Question 3:** In which scenario would Backtracking likely be used?

  A) Finding the shortest path in a graph
  B) Solving optimization problems with weighted choices
  C) Finding all configurations of N queens on a chessboard
  D) Calculating the Fibonacci sequence

**Correct Answer:** C
**Explanation:** Backtracking is suitable for problems like the N-Queens problem, where one must explore all configurations and abandon those that violate constraints.

**Question 4:** Which of the following statements about Greedy Algorithms is true?

  A) Greedy algorithms guarantee the optimal solution for every problem.
  B) Greedy algorithms often yield a locally optimal solution.
  C) They never involve any form of decision-making.
  D) Greedy algorithms are not used for optimization problems.

**Correct Answer:** B
**Explanation:** Greedy algorithms build a solution step-by-step and often yield a locally optimal solution, but not necessarily a globally optimal one.

## Section 6: Analyzing Algorithms

### Learning Objectives
- Understand the concepts of time and space complexity.
- Analyze the performance of algorithms using Big O notation.
- Differentiate between various common time complexities.

### Assessment Questions

**Question 1:** What does Big O notation represent?

  A) The worst-case scenario of an algorithm's performance
  B) The number of lines of code
  C) The programming language used
  D) The security of the algorithm

**Correct Answer:** A
**Explanation:** Big O notation describes the worst-case scenario of an algorithm's performance in terms of time or space.

**Question 2:** Which of the following is an example of linear time complexity?

  A) O(1)
  B) O(n)
  C) O(n^2)
  D) O(log n)

**Correct Answer:** B
**Explanation:** O(n) indicates that the algorithm's execution time increases linearly with the input size.

**Question 3:** What is the space complexity of an algorithm that uses a fixed number of variables regardless of the input size?

  A) O(n)
  B) O(1)
  C) O(log n)
  D) O(n^2)

**Correct Answer:** B
**Explanation:** Using a fixed number of variables results in constant space complexity, represented as O(1).

**Question 4:** In which scenario would you likely prefer using an algorithm with O(log n) complexity?

  A) Searching through a small array
  B) Accessing the first element of an array
  C) Performing operations on a sorted dataset
  D) Sorting a large set of random numbers

**Correct Answer:** C
**Explanation:** O(log n) complexity is common in efficient search algorithms like binary search, especially on sorted datasets.
