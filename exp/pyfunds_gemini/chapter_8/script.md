*   Finally, we have the **boolean** type. This one is simple but incredibly powerful. A boolean can only have one of two values: `True` or `False`. We see `is_beginner = True`. Booleans are the foundation of decision-making in our code, which we'll see in a bit when we talk about control flow. You use them to answer yes-or-no questions like, 'Is the user logged in?' or 'Is the temperature above freezing?'

The key takeaway here is that the data type of a variable determines what operations are valid. You can add two integers, but you can't mathematically add an integer and a string like "hello". Understanding these types is the first step to writing code that works correctly."

**(--- ADVANCE TO FRAME 2 ---)**

"Okay, so now that we know how to store data, how do we interact with it and get new data from the person using our program? That's where operators and user input come into play.

Let's walk through this practical example: we want to ask a user for their age and then tell them how old they'll be on their next birthday.

First, look at this line: `age_str = input("Enter your age: ")`. The `input()` function is our tool for getting information from the user. It does two things: it prints the message inside the parentheses to the screen as a prompt, and then it pauses the program and waits for the user to type something and press Enter.

Now, here is the most important, critical-to-remember detail about the `input()` function: **it always gives us back a string**. Even if the user types the number `25`, Python sees it as the characters '2' and '5', not the numerical value 25. So, what happens if we try to add 1 to the string "25"? We'll get an error, because you can't do math on a piece of text.

This leads us to our next line: `age_int = int(age_str)`. This is where we solve the problem. We perform a process called **type casting**, which is just a fancy way of saying we are converting a value from one data type to another. The `int()` function takes the string `"25"` and converts it into the integer `25`, which we can now use in calculations.

Finally, we have our `print()` statement. Let's break this one down from the inside out. First, we do the math: `age_int + 1`. Since `age_int` is a number, the `+` operator performs mathematical addition. If the user entered 25, this expression evaluates to 26.

But now we have the reverse problem! We want to join the string `"Next year you will be: "` with the number `26`. The `+` operator means something different for strings—it means *concatenation*, or joining them end-to-end. And you can't concatenate a string with a number. So, we have to type cast again! We use the `str()` function to convert the number `26` back into the string `"26"`. The final result is one long string that the `print()` function can display on the screen.

So, to recap the key steps:
1.  We get the data as a string using `input()`.
2.  We **cast** it to an integer to do math.
3.  We use the `+` **operator** to perform the calculation.
4.  And finally, we cast the numerical result back to a string to print it out with our message.

This pattern of getting input, converting types, processing data, and formatting it for output is something you will do constantly as a programmer."

"Alright everyone, let's kick off our midterm review with the fundamental concept of **Control Flow**. Up to now, our programs have mostly been like a straight road—they execute one line after another, from top to bottom. But real-world problems require our programs to make decisions and react differently to different situations. This is where control flow comes in. It allows us to create branching paths and loops in our code."

"Today, we're focusing on the first part: **Conditionals**. This is how we make decisions using the `if`, `elif`, and `else` statements. Think of this as your program asking a series of yes-or-no questions to decide which path to take."

"On this slide, you can see the three main tools for decision-making. At the core of every decision is a **Boolean expression**—something that evaluates to either `True` or `False`, like `score >= 90`."

*   "First, we have the **`if` statement**. This is the entry point. It tests a condition, and if that condition is `True`, the indented block of code right below it is executed."
*   "Next is **`elif`**, which is just short for 'else if.' This is an optional follow-up. It tests a *new* condition, but—and this is key—it only gets checked if the preceding `if` statement was `False`."
*   "Finally, we have the optional **`else` statement**. This is the 'catch-all' at the very end. Its code block runs only if *all* the preceding `if` and `elif` conditions evaluated to `False`. It’s our default path."

"The most critical concept, highlighted in the 'Key Idea' box, is this: Python checks these conditions in order, from top to bottom. As soon as it finds a condition that is `True`, it executes that block of code and then **immediately skips the rest of the entire `if-elif-else` chain.** This means for any single run of this structure, **at most one** code block will be executed."

**(Click to next frame)**

---

**(On Frame 2)**

"To really solidify this, let's trace the example from the previous slide step-by-step. Here, the variable `score` has been set to `75`."

"First, the program gets to the **`if` statement**. It checks the condition: 'Is `75` greater than or equal to `90`?' The answer is **False**. So, what happens? The program completely skips the `print("Excellent!")` line and moves on to the next part of the chain."

"Next, it reaches the **`elif` statement**. It checks this new condition: 'Is `75` greater than or equal to `60`?' This time, the answer is **True**. Because the condition is met, the program executes the indented code block and prints 'Satisfactory.' to the screen."

"And now for the most important part of the trace. Because we found a `True` condition and executed its code, the entire conditional chain is now considered finished. The program **completely skips the final `else` block** without even looking at it. It jumps straight to whatever code might come after this entire structure. This 'one and done' behavior is fundamental."

**(On Frame 3)**

"This top-down, 'one-and-done' logic leads us to a very common point of confusion, and a classic exam-style question: What's the difference between an `if-elif-else` chain and just a series of separate `if` statements? They can look similar, but they behave very differently."

"On the left, we have the **`if-elif` chain**, which we call **mutually exclusive**. The conditions are linked together. If we set `score = 95`, Python checks the first condition: Is `95 >= 90`? Yes, it's true. It prints "Excellent!" and because it's a chain, it immediately skips the rest. The `elif` condition is never even tested. The output is just 'Excellent!'"

"So the key takeaway is this: Use an `if-elif-else` chain when you need to choose exactly one option from a set of related possibilities, like assigning a grade. "Okay, so `for` loops are great when we have a sequence or know the exact number of repetitions. But what about situations where we *don't* know? "A correct `while` loop has three essential parts, which you can see in the example here:"
1.  "First, the **Initialization**. We have to set up our condition variable *before* the loop starts. Here, we initialize `count = 5`."
2.  "Second, the **Condition**. This is the Boolean expression that Python checks at the beginning of *every* iteration. In our case, it's `while count > 0:`. If it's true, the loop body runs; if it's false, the loop terminates."
3.  "And third, the **Update**. This is the most crucial part. Something *inside* the loop must happen that will eventually make the condition false. Here, `count -= 1` decreases the value of `count` by one in each iteration."

"Let's trace the code. Initially, `count` is 5. Is 5 > 0? Yes. So, we print 5 and update `count` to 4. The loop runs again. Is 4 > 0? Yes. Print 4, update to 3. This continues until `count` is 1. "This brings me to a critical warning. What would happen if we forgot the `count -= 1` update step? Can anyone guess?"

**(Pause for student response)**

"Exactly. If `count` starts at 5 and never changes, the condition `count > 0` will *always* be true. The loop will print '5' over and over again, forever. This is called an **infinite loop**, and it's one of the most common bugs for new programmers. Your program will get stuck and you'll have to manually stop it. So, whenever you write a `while` loop, always ask yourself: 'What is happening inside this loop to make it eventually stop?' Make sure you have that update step."

"So, to quickly summarize: `for` loops are for when you know how many times you're looping. `while` loops are for when you're waiting for a condition to become false."

This is useful if you need to remove an item and immediately do something with it. And a quick tip: if you call `.pop()` with no index, it will remove and return the *last* item."

"Finally, let's cover a powerful feature called **slicing**. Slicing lets you extract a portion of a list, creating a new, smaller list from the original."

"The syntax is `[start:stop]`. The most important rule to remember for slicing is that it includes the `start` index but **excludes** the `stop` index. Think of it as 'from the start index, up to, but not including, the stop index'."

"Let's look at the `all_items` list. If we want just 'paper' and 'stapler', we see they are at indices 1 and 2. So, our slice is `all_items[1:3]`. It starts at index 1 and goes up to, but not including, index 3. This gives us a new list: `["paper", "stapler"]`."

"You can also leave out the start or stop. `all_items[:2]` is a shorthand for 'start from the beginning and go up to index 2'. This gives us the first two items: `["pen", "paper"]`."

**(End of final frame)**

"So, to recap, lists are ordered, changeable collections. We use zero-based indexing to access and modify elements, methods like `.append()` and `.pop()` to change the list's size, and slicing to grab sections of the list."

"These operations are the building blocks for almost everything we'll do with collections of data. As we move on to our next topic, **Functions**, you'll see how we can bundle these list operations into reusable blocks of code. This will help us write much cleaner and more powerful programs, following the 'Don't Repeat Yourself' principle."
