### **1. Course Description & Philosophy**

This course is an introduction to the fundamental principles of programming using the Python language. It is designed for students with **no prior programming experience**. We will explore core concepts such as data types, control flow, functions, and data structures. The course places a strong emphasis on developing practical problem-solving skills, learning to write clean and readable code, and building resilience through a systematic approach to debugging.

Our philosophy is that anyone can learn to code. Success in this course comes from active participation, consistent practice, and viewing challenges and errors as learning opportunities. To remove technical barriers, this course uses a **"cloud-first" approach**; all work will be done in a browser-based environment, eliminating the need for local software installation. Assignments are designed to be practical and will draw from a variety of disciplines, reflecting the diverse academic backgrounds of students in the class.

### **2. Learning Outcomes**

Upon successful completion of this course, students will be able to:

*   **Goal 1: Achieve Foundational Programming Competency**
    *   Differentiate between fundamental data types (e.g., integers, strings, lists, dictionaries) and select appropriate types to model problem-specific data.
    *   Construct programs that use conditional logic, iteration, and sequential execution to control the flow of computation.
    *   Decompose programs into modular, reusable functions that accept parameters and return values to manage complexity and improve maintainability.
*   **Goal 2: Cultivate Algorithmic and Problem-Solving Skills**
    *   Analyze a problem statement to identify requirements, inputs, and expected outputs.
    *   Formulate an algorithm using pseudocode or flowcharts to solve a well-defined computational problem.
    *   Implement a given algorithm in Python by translating logical steps into correct and functional code.
*   **Goal 3: Foster Professional and Modern Software Practices**
    *   Write clean, well-documented, and readable Python code that conforms to established style conventions (e.g., PEP 8).
    *   Employ systematic debugging strategies, including the interpretation of tracebacks, to identify, locate, and correct logical and syntax errors in code.
*   **Goal 4: Build a Foundation for Future Learning**
    *   Analyze runtime error messages to diagnose and resolve program faults.
    *   Utilize official documentation and external resources to learn about and apply unfamiliar functions or modules.
    *   Import and integrate components from the Python Standard Library to solve problems more effectively.

### **4. Assessments & Grading Policy**

Your final grade will be determined by your performance on labs, programming assignments, quizzes, and exams. The emphasis is on consistent, hands-on practice.

| Component                 | Weight  | Description                                                                                                                                                                                             |
| ------------------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Weekly Labs**           | 15%     | Held during your scheduled lab section and led by TAs. Graded on completion. Successful completion is defined as making a good-faith effort on all exercises and submitting by the end of the lab session. |
| **Programming Assignments** | 25%     | Weekly assignments submitted to Gradescope. These are the primary tool for developing your programming skills and are auto-graded.                                                                      |
| **Weekly Quizzes**        | 10%     | Short, online quizzes on the LMS to reinforce key concepts from the week's lecture and readings.                                                                                                        |
| **Midterm Exam**          | 15%     | In-class exam covering all topics from Weeks 1-7.                                                                                                                                                         |
| **Final Project**         | 15%     | A culminating project in the final weeks of the course that integrates multiple concepts into a single, larger application.                                                                               |
| **Final Exam**            | 20%     | Cumulative exam covering all course material, with an emphasis on topics from the second half of the semester.                                                                                          |

#### **Late Policy**
You are given **three (3) "late days"** to use on Programming Assignments throughout the semester, no questions asked. Each late day extends an assignment deadline by exactly 24 hours from its original due date/time. A submission made 25 hours late would require two late days. Once your late days are used, assignments submitted late will not be accepted. Late days cannot be used for labs, quizzes, exams, or the final project.
